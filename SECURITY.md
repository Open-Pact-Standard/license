# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in the Open-Pact License text, legal framework, or project infrastructure, please report it responsibly.

## How to Report

1. **Email:** [security@open-pact.dev] (placeholder for actual security email)
2. **GitHub:** Use the [Private Vulnerability Reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) feature for this repository.

## What to Include

* A description of the vulnerability
* Steps to reproduce
* Impact assessment
* Suggested fix (if applicable)

## Response Timeline

* **Acknowledgment:** Within 48 hours
* **Assessment:** Within 7 business days
* **Fix/Patch:** Within 30 days for critical vulnerabilities

## Scope

This policy covers:
* The legal text of OPL-1.1
* The Project Registry infrastructure
* The Canary Token system
* Automated enforcement agents

We do not cover:
* Projects that have adopted OPL-1.1 (report to their maintainers)
* Social engineering attacks outside the project's control

## Safe Harbor

We will not take legal action against security researchers who:
* Follow this security policy
* Do not access, modify, or delete data belonging to others
* Report the vulnerability responsibly
* Do not publicly disclose the vulnerability until a fix is available
